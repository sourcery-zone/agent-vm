# Contributing to Agent VM

First off, thank you for considering contributing to the Agent VM
project! Your help is greatly appreciated.

We welcome contributions in various forms, including:

- Reporting bugs
- Suggesting enhancements
- Improving documentation
- Submitting pull requests with new features or bug fixes

## How to Contribute

### Reporting Bugs

If you encounter a bug, please open an issue on our GitHub
repository. When creating a bug report, please include the following:

- A clear and descriptive title.
- A detailed description of the problem, including steps to reproduce it.
- The expected behavior and what actually happened.
- Your system information (OS, Nix version, etc.).

### Suggesting Enhancements

If you have an idea for a new feature or an improvement to an existing
one, please open an issue on our GitHub repository. Describe your
suggestion in detail, including the motivation and potential use
cases.

## Setting Up a Development Environment

To set up a development environment, you will need to have Nix
installed on your system. For detailed instructions on how to build and
run the VM, please refer to the [Getting Started](#getting-started)
section in the `README.md` file.

Once you have the VM running, you can make your changes to the source
code and test them in the isolated environment.

## Coding Style

This project uses `nixfmt-rfc-style` for formatting Nix code. Please
ensure that your contributions are formatted accordingly. You can use
the following command to format your code:

```bash
nixfmt -f rfc .
```

## Submitting a Pull Request

1.  Fork the repository and create a new branch for your changes.
2.  Make your changes and ensure that the code is properly formatted.
3.  Write clear and concise commit messages.
4.  Push your changes to your fork and submit a pull request to the main
    repository.
5.  In your pull request, describe the changes you have made and reference any
    related issues.

We will review your pull request as soon as possible and provide
feedback. Thank you for your contribution!

## Contact

If you have any questions or need to get in touch with the project
maintainers, please open an issue on GitHub.
